level 256

The reson for this error is the procedure that calculates the bonuses under the maze. When level 256 is reached,it loads the value 255 into the CPU register and incerements it by wan.255 is the maxiumum value that can fit in a Zilog Z80 processor register,so when incremented by 1, an integer overflow occurs and zero is written to the regist

If the register value is less than 7,then the subroutine consider that this is one of the first levels on which to draw bonus symbols. After drawing the register (currendy equal to zero)is decremented by one and a «buttom overflow»ocurs:a negativ number cannot be written in the register ,and 255 is placed in it.The drawing cycle is perfomed with decresing the register value until it becomes equal to 0(i.e time ).The images of bonuses in the game are stored in a special table.Since irs size less than 256,the subroution leaves the table and stars using data that is not interned for this.
